Резюме того, что было реализовано:
DynamicCacheBuster — кастомный ICacheBuster, который позволяет обновлять версию без перезапуска приложения
ThemeEditorPublishedNotificationHandler — отслеживает публикацию страницы themeEditor и автоматически обновляет версию кеш-бастера
SmidgeCacheService — очищает кеш Smidge (физические файлы и IMemoryCache)
Обходной путь в master.cshtml — добавляет query string параметр ?v={version} к URL бандлов для cache busting, не меняя имя файла

Как это работает:
При публикации страницы themeEditor в Production режиме:
Обновляется версия в DynamicCacheBuster
Очищается кеш Smidge
Очищается кеш Umbraco

При загрузке страницы:
Smidge генерирует URL с оригинальной версией (например, .css.v1)
Код добавляет query string параметр ?v={новая_версия}
Браузер видит новый URL и загружает файл
Файл существует по оригинальному пути, поэтому Smidge его находит
